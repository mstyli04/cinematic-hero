# Load-In Assembly + Click Ripple + Translucent Footer — Design

**Date:** 2026-07-03
**Status:** Approved by Styli
**Predecessors:** v2, about-footer, hero-title-live-links specs (all shipped)

## Goal

Three animation "moments": a cinematic opening (particles assemble into
the ring while the name reveals), interactivity (click ripple through the
formation), and a closing image (starfield glowing through a translucent
footer).

## 1. Load-in assembly

**Shader (`particle.vert.glsl`):**
- New `attribute vec3 positionScatter` and `uniform float uAssembly`
  (0 = fully scattered, 1 = assembled).
- After the existing morph mix and BEFORE the ripple and rotation:
  `vec3 assembled = mix(positionScatter, morphed, uAssembly);`
  (subsequent stages operate on `assembled`).

**Geometry (`ParticleField.js`):**
- `positionScatter` generated by the existing `starfield()` shape with
  `spread: 14` (wider than the final starfield's 8, so the fly-in is
  visible from outside the ring).
- New uniform `uAssembly: { value: 0 }` and setter
  `setAssembly(value)` mirroring `setProgress`.

**Load animation (`main.js`):**
- Normal motion: `gsap.to()` on a proxy object driving
  `particleField.setAssembly`, from 0 to 1, `duration: 1.8`,
  `ease: 'power3.out'`, started immediately after the first
  `requestAnimationFrame`.
- Reduced motion: construct with assembly already 1 (call
  `setAssembly(1)` before first render); no tween.

**Name letter reveal (`heroTitle.js`):**
- `createHeroTitle({ animate = true })` — new param, consistent with
  the other UI modules. Markup becomes:
  `<h1 aria-label="MICHAEL STYLIANOU">` containing one
  `<span aria-hidden="true">` per character (space rendered as
  `&nbsp;` inside its span to survive layout).
- `animate: true`: GSAP `from` on the spans — `autoAlpha: 0, y: 24`,
  `duration: 0.8`, `stagger: 0.05`, `ease: 'power2.out'`, `delay: 0.4`
  (letters land while particles are still flying in).
- `animate: false` (reduced motion): spans rendered visible, no tween.
- `setProgress` scroll fade is unchanged (it animates the container
  element's opacity/transform; the letter tween targets the spans —
  no conflict).

## 2. Click ripple

**Shader (`particle.vert.glsl`):**
- New uniforms: `uniform vec3 uRippleCenter;` and
  `uniform float uRippleStart;` (value of `uTime` at click;
  initialized to `-1000.0` so the ripple term is dead until first click).
- After the assembly mix, before rotation:

  ```glsl
  float rippleAge = uTime - uRippleStart;
  float rippleRadius = rippleAge * 3.0;
  float dist = distance(assembled, uRippleCenter);
  float band = exp(-pow(dist - rippleRadius, 2.0) * 4.0);
  float amp = 0.6 * exp(-rippleAge * 1.5);
  vec3 rippleDir = normalize(assembled - uRippleCenter + vec3(0.0001));
  vec3 rippled = assembled + rippleDir * band * amp * step(0.0, rippleAge);
  ```

  (speed 3.0 units/s, band width term 4.0, initial amplitude 0.6,
  decay 1.5/s; `step` kills the term for negative age; the epsilon
  avoids normalize(0).)

**JS (`src/ui/ripple.js`, new module):**
- `initClickRipple({ heroEl, mesh, camera, particleField })`.
- On `pointerdown` on `heroEl`: raycast the pointer through the camera
  (`THREE.Raycaster`) onto the plane `z = 0` (world space), giving a
  world-space point.
- Transform world → shader model space: apply the inverse of
  `mesh.matrixWorld` (covers cursor-parallax rotation), then rotate by
  `+angle` about Y where `angle = uTime * 0.05` (inverse of the
  shader's own `-`…-style rotation — implementer must match the
  shader's rotation sign exactly and verify visually that ripples
  originate under the click at several formation rotations).
- Set `uRippleCenter` to that point and `uRippleStart` to the current
  `uTime` uniform value via a new
  `particleField.triggerRipple(center)` method (reads its own uTime).
- Single active ripple: each click overwrites the uniforms.
- Works for touch taps too (it is a discrete interaction). Reduced
  motion: `initClickRipple` never called.

**Wiring (`main.js`):** inside the existing `if (!reducedMotion)` block.

## 3. Translucent footer (`main.css`)

- `.footer` background changes from `#05060a` to
  `rgb(5 6 10 / 0.85)` — the fixed canvas behind shows the starfield
  (or the static ring under reduced motion) glowing through.
- Text must stay readable: verify contrast visually; if the starfield
  is too loud behind text, deepen to 0.9 rather than adding new
  elements.

## Constraints carried forward

- No new npm dependencies; no changes to renderer.js/deviceTier.js;
  scroll morph contract (uProgress [0,3], clamp 2.9999) untouched.
- Vertex stage order (must hold): morph mix → assembly mix → ripple →
  uTime rotation → parallax happens via mesh.rotation outside the
  shader.
- Attribute count after change: position, A, B, C, D, scatter, phase
  = 7 (WebGL minimum is 16 — fine).

## Verification

Manual/visual via dev server + gstack browser:
- Reload: particles visibly fly in and settle; name letters stagger in;
  after ~2s the scene is identical to today's resting state.
- Click at several points/rotations: ripple visibly expands from under
  the cursor and settles; second click replaces the first.
- Footer: stars visible behind GET IN TOUCH; email/LinkedIn text
  clearly readable.
- Reduced motion: no assembly (instant ring), letters instant, no
  ripple on click, footer readable.
- No console errors; FPS HUD stays near 60 on desktop during ripple.

## Out of scope

Text scramble hover, skills marquee, scroll cue, analytics, deployment.
